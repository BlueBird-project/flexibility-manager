## Coding Style & Constraints
- **Preserve Structure:** Maintain the existing file architecture and directory hierarchy.
- **Naming Parity:** Follow existing variable, function, and data naming conventions strictly.
- **Type Economy:** Do not create new interfaces, types, or classes unless essential for compilation.
- **Minimal Functions:** Prefer slightly longer existing functions over creating new 'utility' functions. 
- **Surgical Changes:** Modify only what is necessary for the task; avoid global refactoring.
